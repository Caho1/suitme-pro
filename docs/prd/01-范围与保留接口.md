
# 范围与保留接口

## 保留原则

只要当前 Java 服务已经对外暴露，默认都不应该轻易删除，除非你已经确认前端完全不消费。

## 当前骨架中保留的接口总数

**81 条**。

其中包括：

- 鉴权与身份：`/login`、`/register`、`/captchaImage`、`/getInfo`、`/logout`
- SUITME 业务接口：`/customer/*`、`/matching/*`、`/matching/tag/*`、`/outfit/*`
- 主数据接口：`/md/product/*`、`/md/productColor/*`、`/md/tag/*`
- 服装分类树：`/md/categoryTag/*`
- 通用接口：`/common/*`
- OSS：`/oss/file/*`
- AI 测试接口：`/api/test/*`
- 系统管理：`/system/user/*`、`/system/role/*`、`/system/menu/*`

## 目前不在骨架中的旧模块

当前骨架没有纳入：

- monitor
- quartz
- tool/gen
- dict/config/post/notice/dept 等非本期确认范围模块

如果你后续确认仍有消费方，可以再加回来。
