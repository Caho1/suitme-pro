
# PRD 总览

## 目标

把 `suitme-java` 后端迁移为 `FastAPI + SQLAlchemy + Alembic`，并尽量保证：

1. 对前端暴露的接口路径不变。
2. HTTP 方法不变。
3. 请求体字段名不变。
4. 响应结构不变。
5. 鉴权方式保持 `Authorization: Bearer <token>`。

## 当前实现策略

当前这份压缩包不是“完整迁移产物”，而是**可以继续开发的基础框架**：

- 基础设施已经搭起来。
- 81 条路由骨架已经就位。
- 文档、目录、脚本、契约清单都已准备。
- 业务逻辑仍需继续逐模块迁移。

## 建议的两阶段路线

### 阶段 A：同库热切换

先保持现有 MySQL 5.7，完成 Java → Python 的服务切换。

### 阶段 B：新数据库迁移

等 Python 版本稳定后，再迁到真正的新数据库。

这样做的原因很简单：

- 如果同时改“语言 + 框架 + 数据库”，风险会叠加。
- 你当前的核心诉求是前端无感热切换，先保持同库最稳。
