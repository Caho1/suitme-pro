
# 数据库与迁移

## 这份 PRD 里最值得补的一点

你的原始诉求里提到了“迁移到新的数据库”，但现有方案同时又写了“第一阶段继续连原 MySQL 5.7”。

这两个目标本身并不矛盾，但必须拆成两个阶段写清楚：

### 阶段 A：服务迁移

- 语言：Java → Python
- 框架：Spring Boot → FastAPI
- 数据库：先继续用现有 MySQL 5.7

### 阶段 B：数据库迁移

- 在 Python 服务稳定后
- 再从现有 MySQL 迁到真正的新数据库
- 新数据库需要明确：类型、版本、字符集、时区、主键策略

## 当前骨架怎么支持这件事

1. 提供了 SQLAlchemy 模型目录。
2. 提供了 Alembic 骨架。
3. 提供了 `scripts/inspect_db.py` 用于先导出真实库结构。

## 为什么不要现在就生成最终 DDL

因为公开仓库里的 SQL 只覆盖了部分表，不能代表现网完整 schema。

所以现在最稳的做法是：

1. 先连接真实库导出完整 schema。
2. 再生成准确的 ORM 与迁移脚本。
3. 再决定新数据库的最终 DDL。
