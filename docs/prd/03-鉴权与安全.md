
# 鉴权与安全

## 当前建议

采用纯 JWT Bearer 模式：

- 登录成功后签发 JWT。
- 后续请求通过 `Authorization: Bearer <token>` 传递。
- 不依赖 Redis 保存登录态。

## `/logout` 为什么建议保留

就算你改成纯 JWT，也建议保留 `/logout`：

- 前端可能已经写死会调它。
- 保留它的成本非常低。
- Python 版里直接返回成功即可。

## 密码哈希

使用 `passlib[bcrypt]`，rounds=10，与 Java 端 `BCryptPasswordEncoder(10)` 保持兼容。

## 必做安全动作

1. 轮换数据库账号密码。
2. 轮换 OSS AccessKey。
3. 轮换 JWT Secret。
4. 所有敏感配置迁到 `.env`。
