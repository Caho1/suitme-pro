
# 接口兼容策略

## 1. 路径与方法不变

所有已保留接口必须保持：

- URL 路径不变
- HTTP 方法不变
- 鉴权头格式不变

## 2. 响应结构保真

### AjaxResult

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {}
}
```

注意：`AjaxResult` 在若依里本质是一个松散的 `HashMap`，所以 Python 端不要把所有响应都绑死成严格模型。

### TableDataInfo

```json
{
  "code": 200,
  "msg": "查询成功",
  "total": 0,
  "rows": []
}
```

### MyBatis-Plus 风格分页

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "records": [],
    "total": 0,
    "size": 10,
    "current": 1,
    "pages": 1
  }
}
```

## 3. 字段风格

- 对外：`camelCase`
- 对内：可以是 `snake_case`

## 4. 时间格式

统一 `yyyy-MM-dd HH:mm:ss`。

## 5. 文件接口也要做契约测试

不能只测 JSON，以下也要测：

- `/common/download`
- `/common/uploads`
- `/common/download/resource`
- `/oss/file/download/{fileId}`
- `/system/user/export`
- `/system/role/export`
