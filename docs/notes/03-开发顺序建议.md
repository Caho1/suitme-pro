
# 开发顺序建议

## 推荐顺序

1. 补全 ORM 模型
2. 打通登录、注册、获取用户信息
3. 完成 `/common/*` 与 `/oss/file/*`
4. 完成 `customer / outfit / matching / tag / product / category`
5. 完成 `system/user`、`system/role`、`system/menu`
6. 补齐契约测试
7. 再考虑新数据库迁移

## 为什么这样排

因为：

- 先打通基础设施，后面所有模块都能复用。
- 文件接口和鉴权最容易影响前端联调。
- 系统管理虽然接口多，但依赖清晰，适合放在业务主链路后面完成。
