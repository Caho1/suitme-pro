
# 仓库风险与注意事项

## 1. 公开仓库的配置风险

当前公开仓库里已经出现了数据库、OSS、JWT 等配置文件。
无论这些值现在还能不能用，都应该立刻视为已暴露并轮换。

## 2. SQL 文件不代表完整现网结构

仓库里有 SQL 目录，但不能直接假定它就是生产数据库的完整真实结构。
尤其是 SUITME 自定义业务表，建议以真实数据库 `information_schema` 导出的结果为准。

## 3. RuoYi 的 `AjaxResult` 很松散

迁移时最容易漏掉的是“额外字段”。
很多接口不是只返回 `{code,msg,data}`，而是还会多塞别的 key。

## 4. 任务状态轮询不要改语义

`digital image` 与 `outfit image` 两类流程里，都有“查本地状态，不够就查外部任务，再回写数据库”的逻辑。
这类时序不能随便重写。
