
# PRD 修订建议

你这份 PRD 主体方向是对的，框架也可以直接用。

但我建议至少补这 5 点：

## 1. 明确拆成两阶段

现在 PRD 同时出现了“热切换”与“迁移到新数据库”两个目标。
最稳的写法应该是：

- 第一阶段：同库热切换
- 第二阶段：新数据库迁移

## 2. 补回 `/logout`

现在 PRD 里没有它，但真实若依前端和很多客户端默认会调用它。
保留它的成本极低，删掉反而有兼容风险。

## 3. 对 `/getRouters` 增加删除前提

不是说这个接口一定要保留，而是要明确：
**只有确认没有任何后台/管理端还在消费时，才能删除。**

## 4. 单独写“文件类契约测试”

你现在 PRD 里更偏向 JSON 接口，但真正容易出问题的是：

- 上传
- 下载
- 预览
- Excel 导出

## 5. 把“敏感配置泄漏整改”写进里程碑

这个仓库已经 public 了，数据库、OSS、JWT 等敏感配置不应该只放在“注意事项”，而应该变成一个必须先做的 M0 动作。
